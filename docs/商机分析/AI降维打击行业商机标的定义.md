# AI 降维打击行业商机 — 标的定义与筛选框架

> 从 9 个细分市场的共性中抽象出的普适性判断模型
> 用于快速评估任何一个行业是否适合用 AI 软件服务切入

---

## 一、核心命题

**AI 降维打击 = 用接近零边际成本的软件，替代高边际成本的人类专家劳动，在价格和可及性上同时碾压现有供给。**

这不是渐进式改良，而是结构性的供给侧重构：

```
传统模式：  客户 ←→ 专家（稀缺、昂贵、不可规模化）
AI 模式：   客户 ←→ AI 系统（无限供给、极低成本、7×24）

价格差：    10x - 100x
可及性差：  从「少数人能用」到「所有人能用」
```

---

## 二、标的必须满足的五个结构性条件

一个行业/场景要成为 AI 降维打击的合格标的，必须**同时**满足以下五个条件。缺任何一个都会大幅降低成功概率。

### 条件一：知识套利空间存在

> **核心交付物是「基于专业知识的判断」，而非物理操作。**

```
✅ 合格：律师审合同、顾问做税务筹划、老师讲题、运营优化 Listing
❌ 不合格：水电工接水管、厨师炒菜、理发师剪发

判断方法：
如果把专家的大脑复制到一个屏幕上，客户能得到 80%+ 的价值 → 合格
如果必须有人亲自到场动手 → 不合格（至少当前阶段不合格）
```

**量化指标**：
- 专家时薪 > 200 元
- 专家产出的核心载体是文字/数据/方案（而非物理动作）
- 客户为「信息/判断/建议」付费，而非为「体力/到场」付费

### 条件二：交付过程可结构化

> **专家的工作可以被拆解为有限步骤的决策树或检查清单。**

```
结构化程度光谱：

高度结构化 ←————————————————→ 高度创意
合同审查    税务筹划    教学    选品分析    品牌策略    艺术创作
  ✅           ✅        ✅       ✅          ⚠️         ❌

AI 擅长替代的位置：左侧 80%
```

**判断方法**：
- 这个工作有没有行业 SOP / 标准流程？→ 有则合格
- 两个同水平的专家做同一件事，结果相似度 > 70%？→ 是则合格
- 能否写出一份详细到初级员工能执行的操作手册？→ 能则合格

**关键区分**：不要求 100% 结构化。只要 70-80% 的工作量可结构化，剩余 20-30% 由人类兜底，就足够构成商业价值。

### 条件三：供需严重失衡

> **客户有需求且愿意付费，但现有供给远远不够——要么太贵，要么太少，要么质量太差。**

```
三种失衡类型（满足任一即可）：

1. 价格失衡：服务存在但太贵
   例：律师审合同 3000 元/份，小企业用不起

2. 数量失衡：服务本身就稀缺
   例：懂 IB 课程的好老师，全国不超过几千人

3. 质量失衡：服务存在但平均质量极低
   例：200 元/月代记账只是走流水，没有任何税务优化
```

**量化指标**：
- 潜在需求方数量 / 现有优质供给方数量 > 100:1
- 优质服务价格 / 普通客户支付能力 > 3x
- 行业 NPS（净推荐值）< 20（客户普遍不满）

### 条件四：效果可感知、可量化

> **用户能在短时间内直观感受到 AI 的价值，而非需要长期信任建立。**

```
感知速度光谱：

即时感知 ←————————————————→ 长期才能验证
Listing优化   报价审计   文书润色   备考刷题   健康管理   品牌建设
出单量↑       省XX万     质量↑      分数↑     指标改善    营收↑
1天见效       5分钟      30秒       1个月     3-6个月    6-12个月
```

**这一条决定了获客难度和留存率**：
- 效果越快可感知 → 获客越容易（免费体验即转化）
- 效果越可量化 → 留存越高（ROI 说话）
- 效果越模糊 → 越依赖品牌和信任 → 获客成本越高

**最佳标的**：用户在首次免费体验中就能感受到价值。
- 「上传报价单 → 能省 XX 万」 ✅
- 「上传文书 → 逐句优化建议」 ✅
- 「输入 ASIN → 生成优化 Listing」 ✅

### 条件五：监管容许空间

> **AI 替代不会触碰行业的法律红线或行政许可要求。**

```
监管风险分级：

绿色（自由进入）：教育辅导、电商运营、内容创作、装修咨询
黄色（需要边界设计）：财税咨询、合同审查、健康管理
红色（强监管，不碰）：医疗诊断、处方开具、法律代理、金融投顾
```

**应对策略**：
- 绿色区域：直接进入
- 黄色区域：**做「信息服务/工具」而非「专业服务」**
  - 不说「我替你审合同」，说「风险提示工具」
  - 不说「我给你诊断」，说「健康信息参考」
  - 不说「我给你做税务筹划」，说「政策匹配建议」
- 红色区域：当前阶段回避

## 三、加分条件（非必要，但显著提升成功率）

### 加分 A：高频重复

```
使用频率越高 → SaaS 订阅模型越成立 → LTV 越高

高频：每天用（教学辅导、电商运营、客服）→ 月订阅
中频：每周/每月用（财税咨询、合同审查）→ 月/年订阅
低频：一次性（装修、留学申请）→ 项目制/按次收费
```

### 加分 B：数据飞轮

```
用户使用 → 产生数据 → 改善 AI → 吸引更多用户

强飞轮：选品数据/录取数据/装修价格数据 → 用越多越准
弱飞轮：通用问答 → 数据对改善帮助有限
```

**有数据飞轮的业务天然有护城河**，即使大厂入局也难以快速追上。

### 加分 C：网络效应或生态效应

```
用户数量本身创造价值：
- 装修 AI：用户越多 → 报价数据越全 → 审计越准
- 留学 AI：用户越多 → 录取数据越全 → 选校越准
- 财税 AI：用户越多 → 案例越全 → 筹划方案越多样
```

### 加分 D：B 端变现可能

```
C 端获客 + B 端变现 = 最佳商业模型

例：
- 装修 AI：C 端免费审报价 → B 端建材商付费获客
- 留学 AI：C 端付费使用 → B 端留学机构购买工具
- 教育 AI：C 端付费 → B 端学校采购
```

### 加分 E：季节性互补

```
如果多条产品线的旺季错开，可以平滑现金流：
- 教育辅导：9 月-次年 6 月（学期）
- 留学申请：8-12 月（申请季）
- 标化考试：全年（考试分散）
- 跨境电商：全年（Q4 旺季）
- 装修：3-5 月、9-11 月（春秋）
```

## 四、评分模型

对任何候选行业，用以下模型快速打分：

### 必要条件（每项 Pass/Fail，任一 Fail 则淘汰）

| # | 条件 | 判断问题 | Pass 标准 |
|---|------|---------|-----------|
| 1 | 知识套利空间 | 核心交付是知识判断而非物理操作？ | 专家时薪 > 200 元且产出为文字/数据 |
| 2 | 可结构化 | 能写出 SOP？两个专家做出来结果相似？ | 70%+ 工作量可标准化 |
| 3 | 供需失衡 | 优质服务太贵/太少/太差？ | 需求方:优质供给方 > 100:1 |
| 4 | 效果可感知 | 用户首次体验就能感受到价值？ | 免费体验 → 可量化价值感知 |
| 5 | 监管容许 | 不触碰强监管红线？ | 绿色或黄色区域 |

### 加分项（每项 0-2 分）

| # | 加分项 | 0 分 | 1 分 | 2 分 |
|---|--------|------|------|------|
| A | 使用频率 | 一次性 | 每周/月 | 每天 |
| B | 数据飞轮 | 无 | 弱 | 强 |
| C | 网络效应 | 无 | 弱 | 强 |
| D | B 端变现 | 纯 C 端 | B 端可能 | C+B 双引擎 |
| E | 与现有业务协同 | 无关 | 技术复用 | 客群重叠 + 技术复用 |

```
总分 = 必要条件全部 Pass + 加分项总分（0-10）

10 分：完美标的，全力投入
7-9 分：优质标的，值得布局
4-6 分：可做标的，需要差异化策略
0-3 分：慎入，除非有独特优势
```

## 五、一句话定义

> **AI 降维打击的合格标的 = 一个由高薪专家用可结构化的方式交付知识判断、但供给严重不足的行业，在这个行业中 AI 能让用户在首次体验时就直观感受到价值，且不触碰监管红线。**

更精炼：

> **贵的、有套路的、稀缺的、见效快的、合法的 — 五个词同时满足，就是 AI 降维打击的标的。**

## 六、用这个框架发现新机会的方法

### 方法一：反向筛查法

```
Step 1: 列出「高薪专业服务」职业
   → 律师、医生、会计、咨询师、设计师、教师、分析师...

Step 2: 逐个检查五个必要条件
   → 律师审合同：Pass（已做 06）
   → 医生诊断：条件 5 Fail（强监管）→ 转为健康管理（07）

Step 3: 对 Pass 的标的跑加分项
   → 排序，选最高分的入场
```

### 方法二：痛点倒推法

```
Step 1: 观察身边人在什么事情上「花大钱但不满意」或「想花钱但找不到」
Step 2: 分析该场景是否满足五个条件
Step 3: 设计「免费体验钩子」测试需求真实性
```

### 方法三：跨行业类比法

```
已验证的模式 → 类比到新行业

「AI 审装修报价单」→「AI 审汽车维修报价单」
「AI 审合同条款」→「AI 审保险条款」
「AI 教学辅导」→「AI 企业培训」
「AI 选品分析」→「AI 选股/选基分析」（注意：条件 5）
```

## 七、终极检验：三个灵魂拷问

在投入开发之前，对每个标的问三个问题：

```
1. 「如果我是这个行业的客户，我会在免费体验后掏钱吗？」
   → 如果你自己都不会 → 放弃

2. 「我能在 4 周内做出一个让人 wow 的 demo 吗？」
   → 如果不能 → 说明结构化程度不够或技术门槛太高

3. 「这个市场的头部玩家 3 年后会不会做同样的事？」
   → 如果会 → 你必须在数据飞轮上比他们早 12 个月
   → 如果不会 → 可能市场太小或他们知道什么你不知道的
```
