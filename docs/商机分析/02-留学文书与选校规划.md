# 留学文书与选校规划 AI — 可行性分析与商业模型方案

> 优先级：P1（立即启动）
> 与国际教育辅导共享客群，开发成本低，客单价高

---

## 一、市场可行性分析

### 1.1 市场规模

| 指标 | 数据 |
|------|------|
| 中国年出国留学人数 | ~80 万人（2025） |
| 使用留学中介/顾问比例 | ~70% |
| 人均留学服务费 | 3-50 万元（按目标国和服务档次） |
| 留学服务市场总规模 | ~600-800 亿元/年 |
| AI 可切入份额（文书+选校） | 核心服务价值占比约 40%，即 240-320 亿元 |
| 3 年 SOM 目标 | 5,000-20,000 万元 |

### 1.2 目标用户画像

**C 端用户**：
- G10-G12 国际学校学生及家长（与 P0 产品高度重叠）
- 大学本科在读（申研）
- DIY 申请者（不想花 10 万+ 请顾问）

**B 端用户**：
- 中小型留学工作室（5-20 人）
- 独立留学顾问
- 国际学校升学指导部门

### 1.3 行业痛点深度剖析

**现有模式的致命问题**：

1. **信息不对称**：顾问掌握的录取数据是核心卖点，但多数顾问经验有限
2. **文书流水线**：大机构一个顾问同时服务 20-40 个学生，文书高度模板化
3. **价格虚高**：高端顾问收费 30-50 万，核心输出就是几篇文书 + 选校列表
4. **质量不可控**：顾问水平差异巨大，学生/家长无法判断
5. **时间瓶颈**：申请季（9-12 月）顾问严重超负荷

### 1.4 竞争格局

| 竞争者 | 模式 | 收费 | 劣势 |
|--------|------|------|------|
| 高端顾问（棕榈/再来人） | 全包服务 | 15-50 万 | 贵、不透明 |
| 中端机构（新东方/启德） | 流水线 | 3-8 万 | 模板化、顾问水平低 |
| DIY 工具（CommonApp） | 自助 | 免费 | 无指导 |
| AI 文书工具（海外） | 润色/生成 | $20-50/月 | 不懂中国学生、无选校 |
| **国内垂直 AI 留学** | **空白** | — | — |

### 1.5 可行性判定

| 维度 | 评分 | 说明 |
|------|------|------|
| 技术可行性 | ⭐⭐⭐⭐⭐ | LLM 天然擅长文书润色/生成 |
| 市场需求 | ⭐⭐⭐⭐⭐ | 刚需，每年 80 万学生 |
| 竞争壁垒 | ⭐⭐⭐ | 数据壁垒需时间积累 |
| 变现确定性 | ⭐⭐⭐⭐⭐ | 家长为升学不惜代价 |
| 监管风险 | ⭐⭐⭐⭐⭐ | 无特殊监管 |

**结论：高度可行，与现有产品协同极强，应立即启动。**

## 二、执行指南

### 2.1 产品架构

```
留学规划 AI 系统
├── 选校引擎 Agent
│   ├── 学生 Profile 分析（GPA/标化/活动/背景）
│   ├── 学校匹配算法（历年录取数据 + 偏好匹配）
│   ├── 选校策略（冲刺/匹配/保底组合）
│   └── 学校详情知识库（排名/专业/文化/录取偏好）
│
├── 文书 Agent 集群
│   ├── 头脑风暴 Agent（挖掘素材和故事线）
│   ├── 文书撰写 Agent（生成初稿）
│   ├── 文书润色 Agent（语言/结构/风格优化）
│   └── 文书审查 Agent（检测 AI 痕迹、查重、对标录取文书）
│
├── 活动规划 Agent
│   ├── 活动列表评估（亮点/短板分析）
│   ├── 活动建议（根据目标学校偏好推荐）
│   └── 时间线规划（按申请季倒推）
│
├── 面试模拟 Agent
│   ├── 学校面试题库
│   ├── 模拟面试对话
│   └── 反馈与评分
│
└── 申请管理
    ├── 进度追踪（各校截止日期）
    ├── 材料清单管理
    └── 一键导出文书
```

### 2.2 分阶段开发计划

```
Phase 1: 文书工具（2026 Q1, 4 周）
├── 文书润色 Agent（上传文书 → AI 优化建议 + 修改）
├── 文书审查 Agent（AI 痕迹检测 + 风格一致性）
├── 支持 Common App / UC / 英国 UCAS 文书格式
└── 独立小程序或 H5 页面

Phase 2: 选校引擎（2026 Q2, 6 周）
├── 学生 Profile 输入系统
├── 选校推荐算法（基于公开录取数据）
├── 选校报告生成
└── 与文书系统打通（根据目标校定制文书策略）

Phase 3: 全流程平台（2026 Q3-Q4）
├── 活动规划 + 时间线
├── 面试模拟
├── 申请进度管理
├── B 端顾问后台
└── 录取数据回收（建立数据壁垒）
```

### 2.3 数据壁垒建设

**最核心的护城河是录取数据**。建设路径：

1. **公开数据采集**：各校 Common Data Set、CollegeBoard 数据
2. **用户数据回收**：用户上传 Profile + 录取结果（以换取免费服务）
3. **合作获取**：与留学机构合作，共享脱敏数据
4. **社区贡献**：建立「录取分享」社区，学长学姐分享 Profile + 结果

目标：3 年内积累 10,000+ 条有效录取记录，形成国内最大的结构化录取数据库。

### 2.4 获客策略

| 渠道 | 方式 | 预估 CAC | 时机 |
|------|------|---------|------|
| 现有教育辅导用户 | 交叉推荐 | ~0 元 | 持续 |
| 小红书 | 「AI 帮我改文书」体验贴 | 30-60 元 | 立即 |
| 知乎 | 留学问答 + 工具推荐 | 40-80 元 | 立即 |
| 留学论坛（一亩三分地等） | 社区种草 | 20-40 元 | Phase 1 |
| B 端顾问推荐 | 给顾问免费额度 | ~0 元 | Phase 2 |
| 学校升学指导合作 | 进校演示 | ~0 元 | Phase 3 |

**核心钩子**：免费文书润色体验（限 500 字，一键对比优化效果）。

### 2.5 关键里程碑

| 时间 | 里程碑 | 验证指标 |
|------|--------|---------|
| 2026 年 4 月 | 文书润色工具上线 | 1,000 次免费体验 |
| 2026 年 5 月 | 付费版上线 | 100 个付费用户 |
| 2026 年 8 月 | 选校引擎上线 | 500 个选校报告生成 |
| 2026 年 10 月 | 申请季高峰 | 2,000 付费用户，MRR 15 万 |
| 2027 年 4 月 | 录取结果回收 | 3,000+ 条录取数据 |

## 三、商业模型方案

### 3.1 收入模型

#### C 端分层订阅

| 套餐 | 价格 | 包含 | 目标用户 |
|------|------|------|---------|
| 免费体验 | 0 元 | 1 篇文书润色（500 字） | 所有人 |
| 文书版 | 199 元/月 | 无限文书润色 + AI 审查 | DIY 申请者 |
| 标准版 | 399 元/月 | 文书 + 选校报告 + 活动评估 | 主力用户 |
| 全能版 | 699 元/月 | 全功能 + 面试模拟 + 进度管理 | 高端用户 |
| 申请季包 | 2,999 元 | 全能版 6 个月 + 不限量 | 申请季一次性购买 |

#### B 端顾问工具

| 模式 | 价格 | 说明 |
|------|------|------|
| 顾问版 | 999 元/月 | 单顾问，管理 20 个学生 |
| 团队版 | 4,999 元/月 | 5 个顾问席位 |
| 机构版 | 按学生数定制 | 白标，学生 200 元/人/月 |

### 3.2 成本结构

| 成本项 | 月均（Phase 1） | 月均（Phase 3） |
|--------|----------------|----------------|
| AI 模型调用（文书生成消耗大） | 5,000 元 | 60,000 元 |
| 云服务 + 向量数据库 | 2,000 元 | 10,000 元 |
| 数据采集 + 清洗 | 3,000 元 | 5,000 元 |
| 运营/内容 | 0 | 20,000 元 |
| 获客 | 3,000 元 | 30,000 元 |
| **合计** | **~13,000 元** | **~125,000 元** |

### 3.3 单位经济模型

```
C 端：
- ARPU = 399 元/月（标准版为主）
- 平均使用月数 = 5 个月（申请季驱动）
- LTV = 399 × 5 = 1,995 元
- CAC = 80 元
- AI 成本/用户/月 = 40 元（文书生成 token 消耗高）
- 毛利率 = (399 - 40) / 399 = 90%
- LTV/CAC = 1,995 / 80 = 24.9x ✅

B 端：
- ARPU = 2,500 元/月（顾问/团队版均值）
- 平均使用月数 = 12 个月
- LTV = 30,000 元
- CAC = 3,000 元（销售拜访）
- LTV/CAC = 10x ✅
```

### 3.4 财务预测

| 指标 | 2026 Q2 | 2026 Q4（申请季） | 2027 Q2 |
|------|---------|------------------|---------|
| C 端付费用户 | 200 | 2,000 | 800（淡季） |
| B 端客户 | 5 | 30 | 50 |
| C 端 MRR | 79,800 | 798,000 | 319,200 |
| B 端 MRR | 12,500 | 75,000 | 125,000 |
| 总 MRR | 92,300 | 873,000 | 444,200 |
| 月成本 | 40,000 | 125,000 | 80,000 |
| 月利润 | +52,300 | +748,000 | +364,200 |

**注意**：留学业务有强季节性（8-12 月为旺季），需要做好淡季现金流规划。

### 3.5 风险与对策

| 风险 | 概率 | 影响 | 对策 |
|------|------|------|------|
| 学校检测 AI 文书并惩罚 | 中 | 高 | 定位为「辅助润色」而非「代写」，内置去 AI 化 |
| 文书质量不够好（top 30 学校） | 中 | 高 | 引入前招生官审核、建立文书评分体系 |
| 季节性收入波动 | 高 | 中 | B 端收入平滑 + 申请季包预收 |
| 数据积累慢 | 中 | 中 | 用免费服务换数据（录取结果回收） |
| 与现有机构冲突 | 中 | 低 | B 端赋能而非取代，做顾问的「AI 助手」 |

## 四、与 P0（教育辅导）的协同

| 协同点 | 具体价值 |
|--------|---------|
| 共享客群 | 辅导用户到 G11 自然转化为留学用户 |
| 共享用户数据 | Mem0 记忆系统积累的学科能力数据可用于选校 |
| 共享技术栈 | 多 Agent 架构、LangGraph、Supabase 完全复用 |
| 交叉获客 | 辅导用户 → 留学规划，留学用户 → 学科辅导 |
| 品牌强化 | 「从辅导到升学」的一站式服务，提高用户生命周期价值 |

**用户旅程**：G5-G9 用辅导 → G10 开始测评 → G11 选校 + 活动规划 → G12 文书 + 申请。
**LTV 从 792 元跃升至 5,000+ 元。**
