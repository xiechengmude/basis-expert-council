# 企业合同 AI 审查 — 可行性分析与商业模型方案

> 优先级：P3
> 高度结构化任务，AI 天然擅长，渗透率极低

---

## 一、市场可行性分析

### 1.1 市场规模

| 指标 | 数据 |
|------|------|
| 中国企业法律服务市场 | ~2,000 亿元/年 |
| 合同审查占比 | ~20-30%（400-600 亿） |
| 中小企业法律服务渗透率 | < 15% |
| AI 合同审查市场（2025） | ~20 亿元，年增长 50%+ |
| 3 年 SOM 目标 | 5,000-20,000 万元 |

### 1.2 目标用户画像

| 用户类型 | 数量 | 合同审查频次 | 当前方案 | 年支出 |
|---------|------|-------------|---------|--------|
| 小微企业 | ~4,000 万 | 10-50 份/年 | 不审/老板自己看 | ~0 |
| 小型企业 | ~300 万 | 50-200 份/年 | 偶尔找律师 | 0.5-2 万 |
| 中型企业 | ~50 万 | 200-1000 份/年 | 法务/外部律师 | 5-30 万 |
| 律所/法务部 | ~5 万 | 批量 | 人工审查 | 人力成本 |

**主力目标**：小型企业（最大痛点）+ 律所法务（效率工具）。

### 1.3 核心痛点

| 痛点 | 详述 |
|------|------|
| 不审查 = 埋雷 | 大量小企业签约不审查合同，事后纠纷频发 |
| 律师贵 | 普通合同审查 2,000-5,000 元/份，复杂的 1-3 万 |
| 审查慢 | 律师通常 3-7 个工作日返回意见 |
| 质量不稳定 | 不同律师关注点不同，容易遗漏 |
| 模板过时 | 企业用的合同模板可能已不符合最新法规 |

### 1.4 竞争格局

| 竞争者 | 代表 | 模式 | 市场份额 |
|--------|------|------|---------|
| AI 合同审查 | 幂律智能/法智易 | SaaS | < 1% 渗透 |
| 传统法律科技 | 法大大/e签宝 | 电子签约 | 合同管理为主 |
| 律所 | 各律所 | 人工审查 | 主流但贵 |
| 通用 AI | GPT 直接审 | 提示词 | 不专业 |

**关键发现**：已有玩家（幂律智能等），但渗透率极低（< 1%），说明市场教育还在早期。

### 1.5 可行性判定

| 维度 | 评分 | 说明 |
|------|------|------|
| 技术可行性 | ⭐⭐⭐⭐⭐ | 合同审查高度结构化，LLM 天然擅长 |
| 市场需求 | ⭐⭐⭐⭐ | 需求存在但需要市场教育 |
| 竞争壁垒 | ⭐⭐⭐ | 先发优势不强，需靠数据和渠道 |
| 变现确定性 | ⭐⭐⭐⭐ | 替代律师，ROI 明确 |
| 监管风险 | ⭐⭐⭐ | 需避免「法律服务」定性 |

**结论：技术上很适合 AI，但市场教育成本较高，竞争也已有先行者。适合作为多产品矩阵的一部分。**

## 二、执行指南

### 2.1 产品架构

```
合同 AI 审查平台
│
├── 合同上传/解析
│   ├── PDF/Word/图片 OCR
│   ├── 合同类型自动识别
│   ├── 条款结构化提取
│   └── 关键信息抽取（甲乙方/金额/期限/违约金）
│
├── 风险审查 Agent
│   ├── 条款风险评分（红/黄/绿）
│   ├── 常见陷阱检测（单方解约权/无限责任/管辖权等）
│   ├── 缺失条款提醒（应有但没有的条款）
│   ├── 模糊表述警告（可能引发争议的措辞）
│   └── 风险等级综合评估
│
├── 条款优化 Agent
│   ├── 逐条修改建议
│   ├── 替代条款推荐
│   ├── 行业最佳实践对标
│   └── 一键生成修改版
│
├── 合同生成 Agent
│   ├── 行业合同模板库（100+ 类型）
│   ├── 自定义条款组装
│   ├── 智能填充（从历史合同提取信息）
│   └── 多版本对比
│
├── 法规匹配 Agent
│   ├── 涉及法律法规自动关联
│   ├── 最新司法解释提醒
│   ├── 类似纠纷案例推荐
│   └── 合规性检查
│
└── 合同管理
    ├── 合同台账
    ├── 到期/续约提醒
    ├── 履约风险监控
    └── 统计分析
```

### 2.2 分阶段开发计划

```
Phase 1: 快速审查工具（6 周）
├── 上传合同 → 5 分钟出审查报告
├── 风险条款标注（红黄绿）
├── 支持 10 种常见合同类型
├── 微信小程序 + Web
└── 核心钩子：免费审查 1 份合同

Phase 2: 深度审查 + 模板（8 周）
├── 修改建议生成
├── 100+ 合同模板库
├── 合同对比功能
├── 法规关联
└── B 端（律所工具）

Phase 3: 合同全生命周期（Q4）
├── 合同台账管理
├── 到期/续约提醒
├── 与 OA/ERP 对接
├── 电子签约集成
└── 行业定制版
```

### 2.3 获客策略

| 渠道 | 方式 | 预估 CAC |
|------|------|---------|
| 「免费审查一份合同」 | 小红书/抖音投放 | 30-80 元 |
| 企业服务平台 | 企查查/天眼查合作 | CPS |
| 律师/法务社群 | 效率工具推荐 | 50-100 元 |
| 企业服务商合作 | 代账公司/人力外包打包推 | 20-50 元 |
| SaaS 生态 | 与 OA/CRM 集成 | API 合作 |

## 三、商业模型方案

### 3.1 收入模型

#### C 端（企业用户）

| 套餐 | 月费 | 包含 |
|------|------|------|
| 免费 | 0 元 | 每月 1 份合同审查 |
| 标准版 | 199 元/月 | 20 份/月 + 模板 |
| 专业版 | 499 元/月 | 无限审查 + 修改建议 + 法规匹配 |
| 企业版 | 999 元/月 | 全功能 + 合同管理 + 多账号 |

#### B 端（律所/法务）

| 模式 | 价格 |
|------|------|
| 律师版 | 599 元/月/席位 |
| 律所版 | 3,999 元/月（10 席位） |
| 法务部版 | 按企业规模定价 |
| API 调用 | 5-20 元/份 |

### 3.2 单位经济模型

```
C 端：
- ARPU = 350 元/月
- 平均留存 = 12 个月
- LTV = 4,200 元
- CAC = 100 元
- AI 成本/审查 = 3-5 元（主要是长文档 token 消耗）
- LTV/CAC = 42x ✅

B 端：
- ARPU = 3,000 元/月
- LTV = 36,000 元
- CAC = 5,000 元
- LTV/CAC = 7.2x ✅
```

### 3.3 财务预测

| 指标 | 2026 Q4 | 2027 Q2 | 2027 Q4 |
|------|---------|---------|---------|
| C 端付费企业 | 200 | 800 | 2,000 |
| B 端客户 | 10 | 30 | 80 |
| 总 MRR | 100,000 | 370,000 | 940,000 |
| 月利润 | +30,000 | +200,000 | +600,000 |

### 3.4 风险与对策

| 风险 | 对策 |
|------|------|
| 被认定为非法法律服务 | 定位「风险提示工具」，明确非法律意见 |
| 审查遗漏导致用户损失 | 免责条款 + 建议用户重大合同仍找律师 |
| 先行者（幂律智能等）优势 | 差异化定位：更便宜 + 更好用 + 与财税工具联动 |
| 长文档 token 成本高 | 预处理 + 分段审查 + 缓存相似条款 |

## 四、与财税 AI 的协同

合同审查与财税顾问面向**同一客群**（中小企业），可以做成**企业服务 AI 套件**：

```
企业 AI 助手套件：
├── 财税 AI 顾问（P3-财税）
├── 合同 AI 审查（本产品）
├── 套餐：企业 AI 管家 699 元/月（含财税+合同）
└── 获客：进入任一产品 → 交叉销售另一个
```

**套餐定价比单独购买便宜 30%，提高客户粘性和 ARPU。**
